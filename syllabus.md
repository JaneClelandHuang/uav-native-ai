# Syllabus

<p class="lede">CSE 40701 · Section 01 · CRN 20469 · Fall 2026</p>

## Learning objectives

By the end of this course, students will be able to:

- Use an AI-assisted development environment (e.g., Claude Code) as a
  disciplined engineering workflow, not an autocomplete shortcut — including
  prompting, architecture design, testing/validation, and systematic review.
- Engineer software components for a cyber-physical system (a UAV) that
  integrates sensing, control, and mission logic, working within an existing
  simulation environment rather than building one from scratch.
- Design and integrate an agentic decision-making layer for autonomous
  behavior.
- Build an operator-facing application supporting safe supervision and
  evaluation of autonomous operations.
- Practice **accountable** AI-assisted development — defend quality claims
  with empirical evidence and appropriate software engineering metrics,
  rather than "it looks right."

## Required tools

- **Docker** (Desktop on Mac/Windows w/ WSL2 backend, or Engine on Linux) —
  runs ArduPilot SITL and the MQTT broker.
- **Python 3.12+** — for the code you'll write and edit directly.
- **Claude Code** (or equivalent AI-assisted development environment).
- **Git / GitHub** — course code and assignments.

No prior UAV or simulation experience assumed. Setup happens in class,
Lesson 1 — see [`code/stage1/SETUP.md`](https://github.com/JaneClelandHuang/uav-native-ai/blob/main/code/stage1/SETUP.md).
